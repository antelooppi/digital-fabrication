Assignment 10: Networking and Communications

Introduction

This week I explored the networking capabilities of the Xiao ESP32C3. I used the test board I made earlier for the input and output devices assignment to write new programs for it. I wanted to write three different codes for it: one to test the connection between the ESP32 and my computer, one to control the servo motor wirelessly, and one to read the encoder value wirelessly. I used same network setup for all of the programs by utilizing ESP32’s STA mode. The computer and ESP32 were both connected to the same network using my home router. I used Aalto AI assistant to help me with the programming, since troubleshooting various different communication problems went quickly beyond my comprehension.

Hello from ESP32!

The first program I wrote was for testing the network connection between the ESP32 and my computer. The program activates a STA interface, and connects it to the router. If the connection succeeds, it prints the ESP32’s IP address to Thonny’s shell so it can be opened in a web browser. My initial plan was to write a minimal test code, using Aalto AI as a teacher to guide me through the process. I planned on using the test code as a basis for adding other functionalities for the oled screen, encoder and servo motor. However, connecting devices over network presented a bunch of challenges I hadn’t anticipated beforehand. For example, the ESP32 had a problem connecting to the network, when restarting the code. It also had some connection issues when powering the device from wall outlet instead of the computer’s usb port. When adding the individual challenges with every input and output device, I quickly realised I didn’t have the time to solve all the problems myself. This was the point where I started to utilise the AI more to write the code instead of helping me to write it.

Sending Data to the Board

For sending data to the board, I wanted to control the servo motor angle wirelessly with the browser. I asked Aalto AI to expand the previous code by adding a html page with the angle adjustment. This value gets send to the ESP32, which changes the angle accordingly. I also asked AI to add the current angle value to the OLED screen for better readability. To make the connection process easier, I asked it to also add code blocks for for AP fallback (if STA isn’t working), clean interface reset before connecting, OLED connection feedback, and bunch of other refinements I don’t understand at this point. With these changes, I was able to run the board both standalone and with computer’s usb port without changing the code or constantly having to restart the board.

Receiving Data from the Board

To receive data from the board, I modified the previous code by removing the servo-related functionalities and adding encoder reading capabilities. I asked AI to use the encoder test code with rotaryirq library I had written for previous assignment as the basis, since I had already solved most of the encoder reading challenges when writing that code. I wanted to be able to read the encoder’s current value with the browser, so I asked AI to do the implementation. I could have achieved the value update in couple ways: manual or automatic. I decided to do the manual version, since the automatic page update would have required some javascript code. The final implementation gets the current encoder reading every time user hits the refresh button in the browser.
