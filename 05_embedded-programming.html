Assignment 05: Embedded Programming

My Interests

I see many possibilities in embedded programming for building audio related tools. For example, I have previously designed and built a synthesizer using Bela board and Puredata, which has been working great. For my next project, I have been thinking of building either a drum machine, a sequencer or a sampler. I think using embedded programming would be a great way to make this build, since I can not only program the logic of the instrument, but also design and build the interface around it. In the future, I would like to use embedded programming for building synthesizer modules, guitar pedals and try it as a basis for interactive sound installations.

Assembling and Testing the RP2040

For this week’s assignment, we used MicroPython to program Seeed Studio XIAO RP2040 microcontroller, which is a cheap and compact development board based on Raspberry Pi RP2040. Alongside with the RP2040, we used Aalto Fablab QPAD.XIAO board, which is a simple board-integrated interface with six capacitive sensing touch pads and a small OLED display. We started the process by soldering the parts together. Here is the list of parts used for the build:

  • 1 x Seeed Studio XIAO-RP2040
  • 1 x Aalto Fablab QPAD.XIAO board
  • 1 x 0.96” OLED module
  • 1 x 3D printed OLED spacer
  • 2 x 10k surface-mount resistors

The soldering process was quite straight forward. I started by soldering two 10k surface-mount resistors to the board. The surface-mount resistors were a lot smaller than what I’m used to, so it was helpful to solder one end at a time by adding some solder to the pad first and then applying the resistor. I used the same approach for attaching the RP2040 module, this time starting from the opposite corners and then applying solder to the rest of the pins. I attached the OLED module to the 3D printed spacer and soldered the screen to the board. Finally, I used a multimeter to test continuity of all connections.

After soldering the board, I installed Thonny editor to my computer, which is an open-source Python IDE for beginners. Before connecting the RP2040 to the computer, I changed Thonny’s settings for Interpreter Device to “MicroPython (Raspberry Pi Pico)” and Port to “Try to detect port automatically”. I then connected the module to the computer using USB cable and installed MicroPython to it. After that, I uploaded a test code from Seeedstudio’s wiki page to the module and ran it. As a result, I was able to get the LED light of the module to blink once every second while Thonny’s shell counted numbers upwards, which was the expected behaviour of the program. After the successful test, I installed external libraries to control the RP2040’s neopixel LED, OLED screen and internal resistor step-response loading measurement. I downloaded the files from our teacher Kris’ Gitlab page

Modifying the Example Programs

I continued by editing a couple of pre-made MicroPython examples, also found from Kris’ Gitlab page. The first file I edited was hello_neopixel.py, which utilizes RP2040’s neopixel LED. This script makes it possible to control the colour of the LED using RGB values, and to program different colours to form a sequence. I edited the script by reducing the number of colors to black and white, and added my own color of orange. I edited the sequence using these three colours. Finally, I changed the speed of the sequence by editing the value of utime.sleep() method.

The second script I edited was hello_oled.py, which displays a static “Hello, world!” text on the display. I wanted to make the text to move, so I began by creating the x, y and frequency variables. After that, I modified the oled.text() method by adding the x and y variables as its coordinates. I wanted the text to move up and down, so I copied the four lines of code in while True section multiple times, and changed their y coordinates by +10 or -10 accordingly. I also added the frequency variable to the sleep() method, so the speed of the movement is easy to change. Of course, as a Finn, I had to also change the text to Finnish, so I created a text variable and added it to the oled.text() method.

My Program

As for my own program, I wanted to utilize the QPAD board’s touch inputs with RP2040’s OLED screen. The bouncing text of the previous assignment reminded me one of the earliest video games ever made, Pong, so I decided to make a simple solo game, where the player tries to keep the ball in the game by moving up and down. The ball bounces back towards the player from the back wall and from the sides. Every time the player succeeds to hit the ball, they get one point added to the score. With every hit, the ball’s speed increases. If the ball passes the player’s character, the game is over.

I used Aalto’s AI Assistant for this assignment. I started by prompting it a similar description of a game idea as above, and added the list of parts I was using. The first versions of the game didn’t work at all, since I had a problem starting the game. I suspected the touch inputs weren’t working correctly, so I asked the AI to make changes to the program based on Kris’ modified hello_touch_neopixel.py patch. This helped a lot, but I also needed to add a description of the QPAD board’s pin arrangement and fine-tune some of the pin numbers inside the code by hand. After these steps, the pads were working, but the physics of the game were acting weird. I prompted couple more suggestions to the AI on how the ball should move, fine-tuned the ball speed, and voilà! – I had a simple Pong-inspired solo game on my QPAD.
