Assignment 08: Electronics Production

My Approach to Electronics Production

The main ways to make PCB boards are chemical etching, CNC milling, and advanced HDI or additive processes that build or remove copper to form the circuit pattern. Chemical etching uses fiberglass‑epoxy boards and chemicals to dissolve copper, CNC milling uses copper‑clad boards and mechanical cutting tools to remove copper, and additive/HDI processes use advanced materials like high‑frequency laminates with lasers and precision plating to create extremely fine features.

For my own practice, I’ll most likely use CNC milling for fast, chemical‑free prototypes I can make myself, while ordering chemically etched boards from an external supplier when I need something more rigid, durable, or suited for mass production.

Preparing the Files for Milling

This week I milled, soldered and coded a test program for the Wolfie v1 board I designed during the last assignment. I started by turning the Gerber files I generated last week into tool paths using CopperCam software. The Gerber is an open industry-standard file system for PCB making. It is basically a blueprint for the PCB manufacturing process, including data for different layers like copper pads, silk screen labels, drill files and such.

CopperCam is able to import Gerber layers and generate customizable tool path sequences with user‑defined drill settings as NC files. For the PCB milling process, the most important tool parameters in CopperCAM are the tool settings for engraving, drilling, hatching, and cutting. The drill bit shape, cutting width, and cutting depth directly determine isolation quality and how much copper is removed. It’s also important to set the correct tool for each machining stage so CopperCAM generates tool paths that match the physical drill bits loaded into the machine. Proper settings and tool selection are what ensure clean, accurate PCB milling.

The Milling Process

I used the Roland SRM-20 to mill the board. I started by measuring the size of my design in KiCad, and picked a board size accordingly. I then measured the thickness of the board, and set it’s value in CopperCam before exporting the NC files. I sanded off the protective coating from the copper surface, and used double-sided tape to attach the board securely to the machine’s wooden plate.

I continued by changing the drill bit to 60 degree v-bit to do the engraving. When changing the drill bit for the SRM-20, the setting of the z origin has to be done manually. This includes changing the drill bit, lowering the drill near the board’s surface (but still leaving room for it to cut), loosening and lowering the drill bit to the working surface by hand, and securing the set screw again. I defined the drill’s Z origin to the position where the drill touched the board’s surface in Roland Vpanel software, which is the software to control the SRM-20. I moved the drill to the lower-left corner of the board and set the XY origin there.

After setting up the drill bit and origin points, it was finally time to mill. I imported the NC files to the Vpanel software and started the cutting process. When the engraving process was done, I changed the drill bit to 0,8mm router, set the Z origin again for the new drill bit, and milled the mounting holes and outlines for the board.

Assembling the Board

After the milling was done, I cleaned the board’s edges and assembled the board. The assembling process was otherwise straightforward, but I had some issues with solder not sticking to the board’s surface. I’m still not sure why this happened, since the test board I had made earlier with similar fashion didn’t have the same problem. I suspected there was still some coating left, so I tried sanding the copper surface more and cleaning it with isopropyl alcohol, but that didn’t help much. I continued soldering the board using a lot of flux, which helped a little. As I had this problem, the soldering process was quite slow and tedious, and I had to do some cleanup work with desoldering wick in couple places. Other than that, I didn’t run into any major issues, and was able to finish the soldering process in few hours. After the soldering was done, I tested the board with multimeter, making sure there was no bridged connections or short circuits.

Writing the Test Code

I wrote a simple test program in MicroPython to verify the circuit by making an LED flash when the button is pressed. I began by importing the necessary pin classes from the machine module and configuring the pins as inputs and outputs. I then added a loop, where the LED remains off by default and turns on only when the button is pressed. Finding the correct pin numbers took some time, as the RP2040 pin diagram was a bit overwhelming to read at first. Once solving the right pins, I was able to successfully light the LED using the button. To make the program more interesting, I also imported the time module and added time.sleep() delay functions in the loop so the LED blinks while the button is held down.

An important accessibility side note to myself and a warning for some readers: The following video contains a flashing light. When having flashing elements on the website, the recommendation is to use slower frequencies than 3 Hz to ensure safety for all viewers (WCAG 2.3.1). I was using a faster sequence in my test code at first, but decided to update it accordingly. I also included a play/stop function to the video and disabled the autoplay function, since any flashing elements can still be distracting or harmful to some people.
